# CL游戏姬 - 开发计划 & 需求文档

## 项目架构

```
acgyx-clone/
├── index.html          # 首页（纯展示，无管理功能）
├── extract-tool.html   # 独立工具：批量提取 acgyx.us 文章 URL 并导出 XLSX
├── admin/
│   └── index.html      # 文章管理后台（含导入、编辑、GitHub 保存）
├── data/
│   └── posts.json      # 所有文章数据（单文件，Git-based CMS）
├── css/
│   └── style.css       # 全局样式
├── js/
│   └── script.js       # 首页逻辑：数据加载、渲染、主题切换
└── functions/
    └── img.js          # Cloudflare Pages Function：图片代理
```

## 部署方式

- GitHub 仓库 `lss5148/CL-YX` → Cloudflare Pages 自动部署
- 推送 `main` 分支后约 1-2 分钟生效
- 管理后台通过 GitHub API（Personal Access Token）直接读写 `data/posts.json`

## 🔜 待实现功能（按优先级排序）

### P0 - 紧急（影响现有功能）

> 当前无 P0 问题。

### P1 - 重要（近期应做）

- [ ] **P1-01 导入并发数上限检测**
  - `importConcurrency` 最大 20，但太多并发可能触发代理限流
  - 方案：在导入弹窗增加提示「建议 3-10」，或在自动检测代理后根据可用代理数量建议并发数

- [ ] **P1-02 GitHub 保存失败后本地数据不丢失**
  - 目前 `saveAll()` 失败后内存中的数据还在，但刷新页面会丢失
  - 方案：将 `postsData` 备份到 `sessionStorage`，在 GitHub 加载失败时从缓存恢复

- [ ] **P1-03 导出导入配置复用**
  - 方案：把导入的 URL 列表、并发数、代理选择保存为可导出的配置文件（JSON），下次直接加载

### P2 - 性能优化（文章超过 200 篇时触发）

- [ ] **P2-01 数据分片存储**
  - 当前：单文件 `data/posts.json`，所有文章一次性加载
  - 问题：超过 500 篇文章时，首页加载变慢，GitHub API 返回文件也可能超时
  - 方案：按分类或月份拆分为多个 JSON 文件
    ```json
    data/
    ├── index.json        # 元数据：分类、标签、分页信息
    ├── posts-001.json    # 文章 1-100
    ├── posts-002.json    # 文章 101-200
    └── ...
    ```
  - 首页只加载 `index.json` + 最新一页的文章数据
  - 管理后台需要改为支持多文件读写

- [ ] **P2-02 首页加载优化**
  - 当前：所有数据在浏览器端渲染，`siteData.posts` 全部加载到内存
  - 方案：首屏只渲染标题 + 封面图，文章卡片正文懒加载
  - 或使用 Cloudflare Pages Functions 做服务端分页，前端按页请求

- [ ] **P2-03 GitHub API 频率限制防护**
  - 认证用户 5000 次/小时，但多次保存失败 + 自动重试会浪费配额
  - 方案：实现请求队列 + 节流，每分钟最多保存 1 次

### P3 - 功能增强

- [ ] **P3-01 文章搜索功能**
  - 首页搜索框目前无实际功能
  - 方案：搜索框触发 `?q=关键词` 参数，renderPosts 按标题/描述/内容关键字筛选

- [ ] **P3-02 文章详情页**
  - 点击文章卡片跳转到 `/article.html?id=N`，但目前 article.html 是静态占位
  - 方案：创建 `article.html`，读取 URL 参数和 `data/posts.json` 渲染完整文章
  - 或使用 Cloudflare Pages Functions 动态生成详情页

- [ ] **P3-03 图片资源本地化**
  - 当前图片 CDN（km-img.top, image.acg.lol, img.dlsite.jp 等）可能被墙或失效
  - 方案：后台增加「图片搬家」工具，自动下载远程图片到本地或自己的图床
  - 方案：增加图片上传功能，上传到 GitHub 仓库或第三方图床

- [ ] **P3-04 多用户支持**
  - 当前只有单 Token 单用户
  - 方案：增加简单的登录系统，区分管理员/普通用户权限

- [ ] **P3-05 操作日志**
  - 记录新增/编辑/删除文章的操作时间和内容摘要
  - 方案：在 `data/posts.json` 增加 `changelog` 字段保存日志

### P4 - 体验优化

- [ ] **P4-01 导入进度优化**
  - 当前进度条按总文章数百分比更新，并行导入时跳动较大
  - 方案：改为按已完成数量平滑更新，显示 ETA 预估剩余时间

- [ ] **P4-02 主题系统完善**
  - 增加更多配色方案选择
  - 方案：预设多套 CSS 变量主题（樱花粉、薄荷绿、深海蓝等）

- [ ] **P4-03 快捷键支持**
  - Ctrl+S 保存、Ctrl+N 新增文章
  - 方案：在 admin/index.html 增加全局键盘监听

- [ ] **P4-04 导入预览**
  - 批量导入前可预览每篇文章的标题和摘要，勾选确认导入哪些
  - 方案：在导入弹窗中增加预览步骤

## 技术债务

| 项目 | 说明 | 影响 |
|------|------|------|
| 文章 ID 自增 | 当前用 `Math.max(...ids) + 1`，删除文章后再新增会复用 ID | 低 |
| 图片代理 | `/img?url=` 依赖 Cloudflare Pages Function，本地开发需额外配置 | 低 |
| 内容安全 | content 使用 `innerHTML` 渲染，如果编辑恶意注入可能 XSS | 低（仅管理员可编辑） |
| 密码硬编码 | 解压密码写在 `posts.json` 中，访问首页即可看到 | 低（非安全敏感） |
